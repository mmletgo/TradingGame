# PRD: NEAT AI 交易模拟竞技场

Created: 2025-12-29
Last Updated: 2026-02-16

## 项目概述

基于 NEAT (NeuroEvolution of Augmenting Topologies) 算法的 AI 交易模拟竞技场。两种类型的 AI Agent（高级散户、做市商）通过神经网络进化算法学习交易策略，在模拟订单簿市场中竞争。噪声交易者提供随机性流动性和布朗运动价格特征。

## 背景与目标

- **背景**: 研究不同类型市场参与者在进化压力下的交易策略演化，观察市场生态的动态平衡
- **目标**:
  - 构建真实的订单簿撮合市场模拟环境
  - 实现两种群独立进化的 NEAT 训练系统
  - 支持多竞技场并行训练，加速进化过程
  - 提供可视化的 UI 演示界面

## 核心业务逻辑

### 1. 市场模拟机制

#### 1.1 订单簿系统
- **深度**: 买卖各100档
- **撮合规则**: 价格优先、时间优先（FIFO）
- **订单类型**: 限价单、市价单
- **价格约束**: 所有操作在最新价 ±100 个最小变动单位内

#### 1.2 杠杆与保证金
| 角色 | 杠杆倍数 | 维持保证金率 | 说明 |
|------|----------|--------------|------|
| 高级散户 | 1.0x | 50% | 无杠杆 |
| 做市商 | 1.0x | 50% | 无杠杆 |

#### 1.3 手续费模型
| 角色 | 挂单费(Maker) | 吃单费(Taker) | 说明 |
|------|---------------|---------------|------|
| 高级散户 | 万2 (0.02%) | 万5 (0.05%) | 普通费率 |
| 做市商 | -万1 (-0.01%) | 万1 (0.01%) | Maker返佣 |
| 噪声交易者 | 0 | 0 | 免手续费 |

### 2. 两种Agent类型 + 噪声交易者

#### 2.1 高级散户 (RetailPro)
- **数量**: 2,400（12子种群×200）
- **初始资金**: 2万
- **信息获取**: 买卖各100档订单簿 + 最近100笔成交 + 最近100个tick历史
- **可用动作**: 挂单买入、挂单卖出、撤单、吃单买入、吃单卖出、不动
- **约束**: 同时只能挂一单

#### 2.2 做市商 (MarketMaker)
- **数量**: 400（4子种群×100）
- **初始资金**: 1,000万
- **信息获取**: 买卖各100档订单簿 + 最近100笔成交 + 最近100个tick历史
- **行为模式**: 每tick必须双边挂单（买卖各1-10单），先撤旧单再挂新单
- **特殊**: 通过报价(QUOTE)而非离散动作决策

#### 2.3 噪声交易者 (NoiseTrader)
- **数量**: 100
- **初始资金**: 1e18（视为无限资金）
- **行为模式**: 每tick以50%概率行动，行动时50%买/50%卖，市价单
- **下单量**: `max(1, int(lognormvariate(mu=3.0, sigma=1.0)))`
- **特殊规则**: 不触发强平，零手续费，可作为ADL对手方

### 3. 强平与ADL机制

#### 3.1 强平流程（三阶段）
强平条件：`保证金率 < 维持保证金率`

**阶段1 - 统一撤单**：
1. 遍历所有Agent检查强平条件
2. 收集需要淘汰的Agent列表
3. 统一撤销这些Agent的所有挂单

**阶段2 - 统一平仓**：
1. 遍历需要淘汰的Agent
2. 提交市价单平仓
3. 收集无法完全成交的Agent（需要ADL）

**阶段3 - ADL自动减仓**：
1. 计算ADL候选清单（包含盈利的噪声交易者）
2. 按排名从高到低选择对手方强制成交
3. 循环直至仓位清零

#### 3.2 ADL排名公式
- 盈利方：`排名 = PnL% × 有效杠杆`
- 亏损方：`排名 = PnL% / 有效杠杆`
- 成交价格：使用当前市场价格（非破产价格）

#### 3.3 淘汰规则
- 被强平的Agent在当前episode剩余时间禁用
- 不参与决策和执行
- Episode结束后通过NEAT进化繁殖新个体

### 4. 噪声交易者机制

#### 4.1 设计目标
- 为市场提供随机性流动性和布朗运动价格特征
- 由中心极限定理保证净买卖量趋向正态分布，实现价格随机游走
- 不参与NEAT进化

#### 4.2 行为规则
- 100个独立噪声交易者
- 每 tick 以 50% 概率行动
- 行动时 50% 买 / 50% 卖，通过市价单撮合
- 下单量服从对数正态分布：`max(1, int(lognormvariate(mu=3.0, sigma=1.0)))`

#### 4.3 特殊规则
- 初始资金 1e18（视为无限资金）
- 不触发强平检查
- 手续费为 0
- 可作为 ADL 对手方
- 有完整的持仓和 PnL 跟踪

### 5. 训练流程

#### 5.1 Episode生命周期
```
初始化
├─ 创建2个种群（从NEAT基因组创建Agent）
├─ 创建100个噪声交易者
├─ 创建撮合引擎
├─ 做市商建立初始流动性（Tick 1）
└─ 初始化EMA平滑价格

Episode循环（N个tick）
├─ 重置所有Agent账户
├─ 重置噪声交易者状态
├─ 重置市场状态
└─ Tick循环
    ├─ Tick 1: 仅做市商行动
    └─ Tick 2+:
        ├─ 强平检查（三阶段）
        ├─ 噪声交易者行动
        ├─ 计算归一化市场状态
        ├─ 随机打乱Agent顺序
        ├─ 并行决策（神经网络推理）
        └─ 串行执行（订单提交）

NEAT进化
├─ 计算适应度（基于PnL）
├─ 执行NEAT进化算法
├─ 清理历史数据防止内存泄漏
└─ 从新基因组创建Agent
```

#### 5.2 多竞技场并行训练
- 每个竞技场独立进程，完全自治
- 通过共享检查点实现物种迁移
- 迁移策略：环形迁移 / 随机迁移 / 最优到最差
- 无锁设计：每竞技场单独写文件

### 6. 神经网络输入输出

#### 6.1 输入构成

| Agent类型 | 输入维度 | 构成 |
|-----------|----------|------|
| 高级散户 | 907 | 200档订单簿(400) + 100笔成交(200) + 持仓(4) + 挂单(3) + 100tick历史(300) |
| 做市商 | 934 | 200档订单簿(400) + 100笔成交(200) + 持仓(4) + 多挂单(30) + 100tick历史(300) |

#### 6.2 输出解析

**高级散户（8个输出）**：
- [0-5]: 动作类型得分（argmax选择）
- [6]: 价格偏移 [-1, 1]
- [7]: 数量比例 [-1, 1]

**做市商（21个输出）**：
- [0-4]: 买单价格偏移
- [5-9]: 买单数量权重
- [10-14]: 卖单价格偏移
- [15-19]: 卖单数量权重
- [20]: 总下单比例

## 用户场景

### 场景1: 单竞技场训练（无UI）
- **用户**: 研究人员
- **操作**: `python scripts/train_noui.py --episodes 100`
- **预期结果**: 高效后台训练，定期保存检查点

### 场景2: 单竞技场训练（带UI）
- **用户**: 研究人员
- **操作**: `python scripts/train_ui.py`
- **预期结果**: 实时可视化训练过程

### 场景3: 多竞技场并行训练
- **用户**: 研究人员
- **操作**: `python scripts/train_multi_arena.py --num-arenas 16 --episodes 100`
- **预期结果**: 16个竞技场并行训练，物种迁移，加速进化

### 场景4: 演示模式
- **用户**: 演示观众
- **操作**: `python scripts/demo_ui.py --checkpoint checkpoints/ep_100.pkl`
- **预期结果**: 实时展示价格曲线、订单簿、成交记录、资产分布
- **结束条件**: 任意物种淘汰到只剩1/4

### 场景5: 恢复训练
- **用户**: 研究人员
- **操作**: `python scripts/train_noui.py --resume checkpoints/ep_50.pkl --episodes 100`
- **预期结果**: 继续之前的进化进程
