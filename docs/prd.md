# PRD: NEAT AI 交易模拟竞技场

Created: 2025-12-29
Last Updated: 2025-12-29

## 项目概述

基于 NEAT (NeuroEvolution of Augmenting Topologies) 算法的 AI 交易模拟竞技场，支持三种类型的 AI Agent（散户、庄家、做市商）通过神经网络进化算法学习交易策略，在模拟市场中竞争。

## 背景与目标

- **背景**: 研究不同类型市场参与者在进化压力下的交易策略演化，观察市场生态的动态平衡
- **目标**:
  - 构建真实的订单簿撮合市场模拟环境
  - 实现三种群独立进化的 NEAT 训练系统
  - 提供可视化的 WebUI 演示界面

## 用户场景

### 场景1: 训练模式（带UI）
- **用户**: 研究人员
- **操作**: 启动训练，实时观察三种群的进化过程
- **预期结果**: Agent 逐渐学会有效的交易策略，实时可视化训练状态

### 场景2: 训练模式（无UI）
- **用户**: 研究人员
- **操作**: 后台运行训练，追求最大性能
- **预期结果**: 高效训练，定期保存检查点

### 场景3: 演示模式
- **用户**: 演示观众
- **操作**: 加载已训练模型，观看模拟交易
- **预期结果**: 实时展示价格曲线、订单簿、成交记录、资产分布

### 场景4: 恢复训练
- **用户**: 研究人员
- **操作**: 从检查点恢复之前的训练
- **预期结果**: 继续之前的进化进程，保持种群状态

## 功能需求

### FR1: 事件引擎
- **描述**: 模块间解耦通信的事件系统
- **优先级**: High
- **验收标准**:
  - [x] 支持发布/订阅模式
  - [x] 所有模块通过事件通信，禁止回调函数
  - [x] 各模块禁止互相 import

### FR2: 日志引擎
- **描述**: 统一的日志管理系统
- **优先级**: High
- **验收标准**:
  - [x] 命令行只显示启动信息和错误
  - [x] 所有日志输出到日志文件
  - [x] 支持按模块分类日志

### FR3: 交易市场模拟引擎
- **描述**: 订单簿撮合交易系统
- **优先级**: High
- **验收标准**:
  - [x] 价格优先、时间优先的撮合规则
  - [x] 支持限价单和市价单
  - [x] 买卖各100档盘口深度
  - [x] 差异化手续费模型（散户万2/万5，庄家0/万1，做市商0/万1）
  - [x] 杠杆交易（散户100倍，庄家/做市商10倍）
  - [x] 强制平仓机制（保证金率低于维持保证金率）
    - 账户检查强平条件：`Account.check_liquidation()`
    - 发布强平事件：`Account.liquidate()` → `LIQUIDATION` 事件
    - 训练引擎/演示引擎订阅事件，提交市价单平仓
    - 撮合引擎撮合成交后，通过 `TRADE_EXECUTED` 事件更新账户
  - [x] 爆仓淘汰并繁殖新 Agent

### FR4: 训练引擎
- **描述**: NEAT 进化训练系统
- **优先级**: High
- **验收标准**:
  - [x] 三种群独立进化
  - [x] 每100 tick 淘汰末尾20%，繁殖前20%
  - [x] 淘汰时打印日志
  - [x] 定期保存检查点
  - [x] 支持恢复训练
  - [x] 可对接演示引擎

### FR5: 演示引擎
- **描述**: WebUI 可视化演示系统
- **优先级**: High
- **验收标准**:
  - [x] 演示模式：加载模型进行模拟
  - [x] 训练模式：实时显示训练过程
  - [x] 显示价格曲线（含成交量）
  - [x] 显示订单簿
  - [x] 显示最近100笔成交
  - [x] 显示三种群资产平均数曲线
  - [x] 显示三种群资产分布小提琴图
  - [x] 每 tick 更新界面
  - [x] 支持暂停/恢复/重置

### FR6: 生物系统
- **描述**: 三种类型的 AI Agent
- **优先级**: High
- **验收标准**:
  - [x] 散户：挂单/撤单/吃单/不动（同时只挂一单）
  - [x] 庄家：挂单/吃单（绝不不动，不能单纯撤单，同时只挂一单。下单时若有挂单则先自动撤单）
  - [x] 做市商：双边挂单/清仓（每 tick 必然双边挂单，每次先撤掉所有旧单再挂新单，每边 1-10 单在不同价位）
  - [x] 所有操作在最新价 ±100 个最小变动单位内
  - [x] 初始化时只有做市商行动
  - [x] 后续按 做市商→庄家→散户 顺序，同物种并行

## 非功能需求

### 性能
- 优先使用向量化操作（NumPy）
- 无法向量化的场景使用 Cython 加速
- 订单簿撮合使用 Cython 实现

### 可配置性
- 所有参数可在配置文件中配置
- 支持 NEAT 配置文件自定义

### 可维护性
- 模块解耦，通过事件通信
- 严格的 Python 类型定义

## 配置参数

### 市场参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 初始价格 | 100 | 交易品种初始价格 |
| 最小变动单位 | 0.1 | 价格 tick size |
| 最小交易单位 | 1 | 数量 lot size |
| 盘口深度 | 100 | 买卖各100档 |

### 手续费参数
| 角色 | 挂单(Maker) | 吃单(Taker) |
|------|-------------|-------------|
| 散户 | 万2 (0.02%) | 万5 (0.05%) |
| 庄家 | 0 | 万1 (0.01%) |
| 做市商 | 0 | 万1 (0.01%) |

### 杠杆参数
| 角色 | 杠杆倍数 | 维持保证金率 |
|------|----------|--------------|
| 散户 | 100 | 0.5% |
| 庄家 | 10 | 5% |
| 做市商 | 10 | 5% |

### 生物参数
| 角色 | 数量 | 初始资产 |
|------|------|----------|
| 散户 | 10000 | 1万 |
| 庄家 | 10 | 1000万 |
| 做市商 | 100 | 1000万 |

### 训练参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 淘汰间隔 | 100 tick | 每隔多少 tick 进行淘汰 |
| 淘汰比例 | 20% | 淘汰末尾比例 |
| 繁殖比例 | 20% | 使用前多少比例繁殖 |

## 技术栈

- **后端**: Python 3.11+, FastAPI
- **前端**: React + TypeScript + ECharts
- **NEAT**: neat-python
- **加速**: NumPy, Cython
- **通信**: WebSocket
- **持久化**: pickle
